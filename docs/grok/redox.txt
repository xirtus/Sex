# SEX SASOS Architecture Handoff: Pure Rust Migration Plan

**Document Status:** Active
**Target Architecture:** `x86_64-unknown-none`
**Core Objective:** Eradicate legacy C dependencies across all major subsystems to maintain a pristine, lock-free Trusted Computing Base (TCB). This plan outlines the strict `no_std` migration path for hardware initialization, cryptography, networking, and the boot sequence.

---

## 1. Bootloader and The Handoff
**Current Bottleneck:** Traditional x86_64 boot paths rely on C/ASM glue code to translate memory maps and system states into a format the kernel can understand.
**The Pure Rust Strategy:**
* **Implementation:** Utilize the `limine` Rust crate to handle the boot protocol entirely in native code.
* **Execution:** * Remove all C-based trampoline code from the entry sequence.
    * Configure the kernel to directly consume the memory map, Root System Description Pointer (RSDP), and Symmetric Multiprocessing (SMP) core data directly into safe, native Rust structs at the exact moment of entry.
    * This guarantees memory safety guarantees begin at instruction zero.

## 2. Hardware Discovery and Initialization (ACPI/PCIe)
**Current Bottleneck:** Monolithic C codebases like ACPICA require extensive OS-specific mutexes and memory allocators just to read hardware tables.
**The Pure Rust Strategy:**
* **Implementation:** Integrate the `acpi` and `aml` crates.
* **Execution:**
    * Implement a 100% `no_std` parser to scan the RSDP.
    * Parse the MADT (Multiple APIC Description Table) and MCFG (Memory Mapped Configuration Space Base Address Description Table) without linking any C libraries.
    * Use this native discovery phase to map PCI buses and initialize the NVMe controllers for the Alienware hardware before shifting operations into the zero-copy hot paths.

## 3. Cryptography and TLS
**Current Bottleneck:** Standard crypto libraries aggressively pull in POSIX threads (`pthread`) and standard C libraries (e.g., `aws-lc-sys`, `ring`), breaking the `unknown-none` target constraints.
**The Pure Rust Strategy:**
* **Implementation:** Standardize on the **RustCrypto** organization crates and a neutered `rustls`.
* **Execution:**
    * For hashing, signatures, and basic encryption, deploy crates like `sha2`, `ed25519-dalek`, and `chacha20`. These compile perfectly to bare metal.
    * For Transport Layer Security (TLS) within the server architecture, implement `rustls`.
    * **Critical Constraint:** The default cryptography provider in `rustls` must be explicitly repointed to a pure-Rust backend (using the `rust-crypto` feature flags) to forcefully strip `aws-lc-sys` and its C-bindings from the dependency tree.

## 4. Network Stacks (TCP/IP)
**Current Bottleneck:** Porting highly optimized C stacks like lwIP requires unsafe FFI boundaries and risks heap corruption inside the lock-free architecture.
**The Pure Rust Strategy:**
* **Implementation:** Deploy `smoltcp`.
* **Execution:**
    * Integrate the `smoltcp` standalone, event-driven TCP/IP stack.
    * Because it requires no standard library and spawns zero native threads, bind its event loop directly to the custom lock-free IPC signal framework.
    * Map network packet buffers directly into the single address space for zero-copy routing between the network driver and userland services.

### Script #1 – `cargo xtask pure-rust-v23` (the heavy lifter)  
Create directory `xtask/` (standard Rust convention) and add this file as `xtask/src/main.rs`:

```rust
// xtask/src/main.rs
use std::fs;
use std::process::Command;

fn main() {
    println!("🚀 SEX Pure-Rust v2.3 Migration – executing...");

    // 1. Update kernel/Cargo.toml with exact 2026 pins
    let cargo = include_str!("../../kernel/Cargo.toml");
    let new_cargo = cargo
        .replace(r#"limine = ".*""#, r#"limine = "0.6.3""#)
        .replace(r#"[dependencies]"#, r#"[dependencies]
acpi = "6.1.1"
aml = "0.16"
pci_types = "0.10"
nvme-oxide = "0.4"
smoltcp = { version = "0.13.0", default-features = false, features = ["proto-ipv4", "proto-ipv6", "socket-tcp", "socket-udp", "log"] }
rustls = { version = "0.23", default-features = false }
rustls-rustcrypto = "0.3"
sha2 = { version = "0.10", default-features = false }
chacha20poly1305 = { version = "0.10", default-features = false }
ed25519-dalek = { version = "2.1", default-features = false }
x2apic = "0.10"
x86_64 = "0.15""#);
    fs::write("kernel/Cargo.toml", new_cargo).unwrap();

    // 2. Patch kernel/src/main.rs with pure limine _start
    let main_rs = include_str!("../../kernel/src/main.rs");
    let new_main = main_rs
        .replace("entry_point!", r#"#[link_section = ".requests"]
#[used]
static LIMINE_REQUESTS: limine::Requests = limine::Requests::new([
    limine::BaseRevision::new(),
    limine::HhdmRequest::new(),
    limine::MemoryMapRequest::new(),
    limine::RsdpRequest::new(),
    limine::SmpRequest::new(),
]);

#[no_mangle]
pub extern "C" fn _start() -> ! {
    pdx::bootstrap();
    gemini::spawn_self_repair_thread();
    // … rest of kernel
    loop {}
}"#);
    fs::write("kernel/src/main.rs", new_main).unwrap();

    // 3. Create missing pure-Rust modules (hw/init.rs, crypto/pdx.rs, etc.)
    fs::create_dir_all("kernel/src/hw").unwrap();
    fs::write("kernel/src/hw/init.rs", include_str!("templates/hw_init.rs")).unwrap(); // (templates/ can be added once)
    // … repeat for alloc.rs, network/pdx.rs, gemini/crate_freshness.rs

    // 4. Add .github/renovate.json
    fs::write(".github/renovate.json", r#"{
  "packageRules": [
    {
      "matchPackageNames": ["limine", "acpi", "smoltcp", "rustls*", "nvme-oxide"],
      "rangeStrategy": "bump",
      "prBody": "sex-gemini validation required"
    }
  ],
  "postUpdateOptions": ["cargo:check"]
}"#).unwrap();

    println!("✅ All files patched. Running cargo check + sex-gemini validation...");
    Command::new("cargo").args(["check", "-p", "sex-kernel"]).status().unwrap();
    // sex-gemini self-repair hook
    Command::new("./run_sasos.sh").arg("--validate-pure-rust").status().unwrap();
}
```
